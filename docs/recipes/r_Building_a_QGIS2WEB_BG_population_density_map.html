Objectives

  • This project aims to create a comprehensive choropleth map visualizing the population density statistic for Bulgaria in an interactive environment that can be viewed online.
  • The project strives to fulfill the following objectives:
    • The data used will be official government data or free open-sourced data available online. All data sources will be documented in the project description.
    • The data used will be continuously checked for consistency against official sources. Discrepancies will be documented.
    • The exported map will illustrate the population density statistic at zemlishte/settlement ground level.
    • The exported map will contain the statistic for all settlements contained in a single zemlishte correctly aggregated and ranked.
    • The exported map will contain an on-click annotation layer to display data for the settlements included in a given zemlishte.

Prerequisites

Software

  • QGIS
  • DBeaver or other software to query SQLite databases.
  • QGIS2WEB plugin installed and working for QGIS.

Input Data

  • For this project we are going to use the following data sources:
    • Bulgaria-geocoding (Administrative boundaries for Bulgaria).
    • Clean-Stats (Compiled statistical information for the administrative units).

QGIS Project Preparation

Working Directory

  • Prepare a working directory for the project.
  • Clone the input data repositories in the input/ directory.

Project Setup

  • Start QGIS, create a new project, and save it in the working directory.
  • In Project -> Properties:
    • Add project title: Bulgaria Population Density 2018.
    • Set units for distance measurement to kilometers.
    • Set units for area measurement to square kilometers.
  • Fill in the Metadata section.
  • Change coordinate system to: WGS 84 / UTM zone 34N. (note: optional, will not export).

Data Preparation and Verification

Population Data File

  • We will extract the settlements population data for 2018 from the Clean-Stats project and will save it as a csv file to work with later.
  • Prepare the population data file:
    • In DBeaver open the clean-stats database.
    • Use the query below to export the settlements population information needed:
    • Save in input/ as census_data_settlements_2018.csv
SELECT EKATTE_ID, POPULATION FROM SETTL_POPULATION_NSI 
WHERE START_DATE <= date('2018-01-01') AND END_DATE > date('2018-01-01')

Verify Data

  • Check the number of settlements in EKATTE official source: 5256.
    • National Register of Populated Places URL: https://www.nsi.bg/nrnm/
  • Check the number of settlements in Clean-Stats database with the query below: 5256.
SELECT COUNT(*) FROM settlements 
WHERE START_DATE >= date('2018-01-01') and END_DATE < date('2018-01-01');
  • Check the number of settlements in Bulgaria-geocoding settlements.csv file with the expression below: 5256.
count_distinct( "EKATTE_ID" )
  • Verify the number of grounds features:
    • The number of features in the grounds.geojson file: 4615.
    • Check the number of settlements with parent grounds in settlements.csv with the expression below: 641.
count( "GROUND_PARENT_ID" )
  • Assure that the total matches the number of settlements from above: 4615 + 641 = 5256.
  • Assure that the EKATTE_ID column is type text for all used layers, and the data columns are numeric to prevent the need to cast types later in the process.
  • Verify if the settlements.csv has GROUND_PARENT_ID applied for all features. Settlements with own grounds should have their EKATTE_ID as a GROUND_PARENT_ID.
  • Verify if the fields for numeric attributes (e.g. area in settlements.csv) are loaded with the proper type or numeric formulae won't be applicable for these fields.

Defining Data Relationships

  • Since multiple settlements can belong to a single ground polygon entity we need to define a many-to-one relationship between them. That will allow us to compute aggregate values directly in QGIS:
    • In Project -> Properties -> Relations click on Add Relation.
    • Add relation name: rel_ground_settlements.
    • The parent is Grounds with a referenced key EKATTE_ID.
    • The referencing layer (child) is Settlements with referencing field GROUND_PARENT_ID.
    • Name the Id for convenience: rel_sg.
    • The relationship strength is Association.
  • The settlement polygons now have a one-to-many relationship to the look aside table rel_settl_ground established.

Defining Settlements-Population Table Join

  • In the Settlements layer create a join to the Population 2018:
    • Join field: EKATTE_ID. Target field: EKATTE_ID.
    • Enable Custom Field Name Prefix: pop2018_.
    • Settlements should now have a pop2018_POPULATION column of integer type.
  • Verify if the population records have successfully joined.
    • Note the sum of all matched features' individual population records: use sum("pop2018_POPULATION").
    • Sum is 7,000,039. The records from the population layer have successfully joined the Settlements layer.

Computing Required Data

Add Styling Functions

  • A couple of python functions that we will use to prepare annotation templates.
  • In the Field Calculator's Function Editor add:
    • Function to create annotations' data:
# python 3.6+
from qgis.core import *
from qgis.gui import *

@qgsfunction(args='auto', group='Deo_FmtFunc')
def pop_density_map_create_event_annotations(data_year,
                                             ent_fullname,
                                             pop_density,
                                             pop_total,
                                             ent_area,
                                             children_ids,
                                             children_names,
                                             children_pop,
                                             feature, parent):
    children_ent_info = list(zip(children_ids.split('|'),
                                 children_names.split('|'),
                                 children_pop.split('|')))
    # Sort by population
    children_ent_info.sort(key=lambda tup: int(tup[2]), reverse=True)
    anno_data = {
        'data_year': data_year,
        'ent_name': ent_fullname,
        'pop_density': pop_density,
        'pop_total': pop_total,
        'ent_area': ent_area,
        'child_list': children_ent_info,
    }
    return anno_data
  • Function to create HTML event annotations:
# python 3.6+
@qgsfunction(args='auto', group='Deo_FmtFunc')
def pop_density_map_format_event_annotations(anno_data, feature, parent):
    ch_ent_rows = []
    for child in anno_data['child_list']:
        ch_row = f'<tr><td><strong>{child[0]}</strong></td><td>{child[1]}</td><td>{child[2]}</td></tr>'
        ch_ent_rows.append(ch_row)

    ch_ent_table = f"""
<table>
    <tr><th>ID</th><th>Населено място</th><th>Брой жители ({anno_data['data_year']})</th></tr>
    {''.join(ch_ent_rows)}
</table>
"""

    event_anno_html = f"""
<h4>{anno_data['ent_name']}: {anno_data['pop_density']} д/км2</h4>
<hr>
<h5>Население: {anno_data['pop_total']}</h5>
<h5>Площ: {anno_data['ent_area']} км2</h5>
<h5>Населени места в обхвата на землището: </h5>
{ch_ent_table}
"""
    return event_anno_html

Data Fixes and Enhancement

  • Use the field calculator with the Grounds layer to create all of the columns detailed below.
    • Don't create virtual columns, it slows everything down, use permanent columns.
    • After each column is created stop editing and save changes to the layer (I had a weird table update issue, possibly due to using geojson format).
  • Create column AREA_KM2.
    • Use: relation_aggregate('rel_sg', 'sum', area) / 1000
    • Confirm the calculation is correct by summing the AREAKM2 column to the total area of the country.
  • Create column SETTL_COUNT.
    • Use: relation_aggregate('rel_sg', 'count', EKATTE_ID)
    • Confirm the calculation using sum("SETTL_COUNT") to check the total settlement count.
    • If the result matches the total amount of settlements in Bulgaria we can conclude the relationship information provided is correct.
  • Create column POP_NSI_2018:
    • Use: relation_aggregate('rel_sg', 'sum', pop2018_2018).
    • Confirm the calculation is correct by summing the total to the previously produced figure for total CA population.
  • Create column POP_DENSITY_NSI_2018:
    • Type is Decimal with precision 2.
    • Use: round("POP_NSI_2017"/"AREA_KM2" , 2)
  • Create column SETTL_NAMES_LIST:
    • Use: relation_aggregate('rel_ground_settlements', 'concatenate', "t_v_m" || ' ' || "name", '|')
  • Create column SETTL_ID_LIST, a new text field with the IDs of all settlements belonging to particular grounds polygon:
    • Set the output field length to 0.
    • Use: relation_aggregate('rel_ground_settlements', 'concatenate', EKATTE_ID, '|')
  • Create column POP_NSI_2018_LIST:
    • Set the output field to text and the output field length 0.
    • Use: relation_aggregate('rel_ground_settlements', 'concatenate', coalesce(to_string("pop2018_POPULATION"), 'N/A'), '|')
  • Create POP_DENSITY_NSI_2018_EVENT_ANNO, a label column to hold the string to display on hover (or on click).
    • Use text output field type with field length 0.
    • Use the annotation creating formulae:
pop_density_map_format_event_annotations( (pop_density_map_create_event_annotations('2018',  "SETTLEMENT_FULLNAME" ,  "POP_DENSITY_NSI_2018" , "POP_NSI_2018" , "AREA_KM2" , "SETTL_ID_LIST" , "SETTL_NAMES_LIST" , "POP_NSI_2018_LIST" )))

Map Background

    • 1-2

    • 30-60

    • 480-1000

  • Apply color ramp Viridis against the defined classes.

Grounds' Borders

  • Set the Grounds layer's polygon boundary as 0.1mm color: #5a5a5a

Map Annotations

  • Event annotations on maps exported via qgis2web (on hover, on click events) can be controlled via the Form settings window.
  • Prepare Grounds layer's event annotations:
    • Rename the layer to Землища.
    • Set all fields' widgets to "hidden".
    • Set the POP_DENSITY_NSI_2018_EVENT_ANNO widget type to Text Edit. Check Multiline and HTML options.
    • Verify the rendered HTML.

Export Using QGIS2WEB

  • Select the OpenLayers export option.
  • Layers and Groups:
    • Turn off Popups for Bulgaria and Grounds layers.
  • Appearance:
    • Set Add layers list to Collapsed.
    • Select Template to full-screen.
    • Set Min zoom level to 7.
    • Set Max zoom level to 11.
    • Check the Restrict to extent option.
  • Data export:
    • Set the Exporter to Export to folder and choose the directory.
  • Check the result preview and export.
